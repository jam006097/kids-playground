# ポートフォリオPDF生成機能 構築計画書

## 1. 目的
本プロジェクト「kidsPlayGround」の成果を視覚的に提示するため、ローカル開発環境で動作する主要なWebページを自動的に巡回し、スクリーンショットを撮影、それらを1つのPDFファイルにまとめて出力するDjangoカスタム管理コマンドを開発する。

## 2. 技術選定
*   **ブラウザ自動化:** `Playwright for Python`
    *   理由: モダンなレンダリング（Flexbox/Grid）の再現性が高く、PDF出力機能が標準で備わっているため。
*   **PDF結合/編集:** `pypdf`
    *   理由: Playwrightで生成した個別のPDFページを1つに統合するため。
*   **認証処理:** `Playwright` の認証コンテキスト保持機能
    *   理由: ログインが必要なページ（マイページなど）も自動で撮影するため。

## 3. 開発方針（GEMINI.md準拠）
*   **TDD (Test Driven Development):**
    *   `Red -> Green -> Refactor` のサイクルを遵守。
    *   テスト名は日本語で「〜のとき〜であること」と記述する。
*   **SOLID原則:**
    *   `Command` クラス（実行制御）、`BrowserOperator`（ブラウザ操作）、`PdfComposer`（PDF結合）に責務を分離する。
*   **保守性:**
    *   対象URLリストは拡張性を考慮し、別定義とする。

## 4. 作業フェーズとタスク

### Phase 1: 環境準備
* [ ] 依存ライブラリ (`playwright`, `pypdf`) の追加
* [ ] Playwrightのブラウザ（Chromium）のインストール確認

### Phase 2: 設計と調査
* [ ] `urls.py` を解析し、撮影対象のページリストを作成（一覧、詳細、ログイン、マイページ、お気に入り等）
* [ ] ログインが必要なページへのアクセス戦略の策定

### Phase 3: 基盤実装 (TDD)
* [ ] 単一URLをPDF化する機能のテストと実装
* [ ] 自動ログイン処理のテストと実装

### Phase 4: コマンド実装
* [ ] Django管理コマンド `generate_portfolio_pdf` の作成
* [ ] URLリストの巡回と一時保存ロジックの実装
* [ ] 全一時ファイルを統合するPDF結合ロジックの実装

### Phase 5: 検証と調整
* [ ] 出力PDFのレイアウト確認（ヘッダー、フッターの有無など）
* [ ] エラーハンドリング（サーバー未起動時の対応など）

## 5. 実行ルール
* **セキュリティ:** 認証情報は `.env` から取得し、ログに出力しない。
* **クリーンアップ:** 結合後の中間ファイル（画像等）は必ず削除する。
* **品質管理:** `flake8`, `mypy` によるチェックをパスさせる。
